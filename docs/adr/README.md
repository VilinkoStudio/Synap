# 架构决策记录

这个目录用于保存 Synap 的 Architecture Decision Record（ADR）。

## 索引

| ADR | 标题 | 状态 |
|-----|------|------|
| 0001 | core 保持平台无关，网络与发现能力外置 | 已接受 |
| 0002 | 明确 corenet 范围与 core 中普通 HTTP 能力边界 | 已接受 |
| 0003 | 引入零信任 relay 作为去中心化同步的补充 | 已接受 |

## 状态说明

- `提议中`：还在讨论，尚未定案
- `已接受`：已经确认，后续实现与重构遵循该决策
- `已废弃`：曾经有效，但现在不再适用
- `已替代`：被新的 ADR 替代
- `已拒绝`：认真评估过，但最终不采用

## 编写约定

1. 用简洁文字写清楚背景、约束、备选方案、最终决策与后果。
2. 已接受的 ADR 不直接改结论；如需改变，新增一条 ADR 覆盖它。
3. 优先记录真正会影响边界、职责、扩展方式的设计决定，而不是实现细节。
